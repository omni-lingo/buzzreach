# ATOM: BILL-004 — Customer Portal (Billing & Subscription Management)

**Layer:** L4
**Module:** billing
**Effort:** M
**Depends on:** BILL-002, BILL-003

## Inputs (what this atom reads/consumes)
- `src/backend/models/subscription.py` — subscription data
- `src/backend/services/stripe_service.py` — Stripe API
- `src/backend/models/usage.py` — usage tracking

## Outputs (what this atom produces)
- `src/frontend/pages/BillingPortal.tsx` — portal showing:
  - Current plan (name, price, features)
  - Upgrade/Downgrade button
  - Usage this month (progress bar)
  - Billing cycle dates (start, renewal, trial ends)
  - Payment method (last 4 digits of card)
- `src/frontend/pages/InvoiceHistory.tsx` — invoices:
  - Table of past invoices (date, amount, status, download link)
  - Filter by date range
  - Download invoice as PDF
- `src/frontend/pages/PlanComparison.tsx` — upgrade flow:
  - Side-by-side plan comparison (free/pro/premium)
  - Current plan highlighted
  - Upgrade button → Stripe checkout
  - Downgrade option (with proration explanation)
  - Cancel subscription button (with retention offer)
- `src/backend/api/billing.py` — routes:
  - GET `/api/v1/billing/current` — current plan + usage
  - GET `/api/v1/billing/invoices` — invoice list
  - POST `/api/v1/billing/upgrade` — initiate upgrade
  - POST `/api/v1/billing/downgrade` — initiate downgrade
  - POST `/api/v1/billing/cancel` — cancel subscription (with survey)
- Stripe portal integration (customer.stripe.com linked):
  - Or build custom portal UI
- `tests/test_billing_portal.tsx` — navigate, upgrade, cancel

## Acceptance criteria
- [ ] Current plan displayed correctly
- [ ] Usage bar shows accurate percentage
- [ ] Upgrade button links to Stripe checkout (or stays in app)
- [ ] Downgrade shows proration calculation
- [ ] Invoice download works (PDF generated by Stripe)
- [ ] Payment method shows (not full card number)
- [ ] Cancel button shows retention offer ("1 month free?")
- [ ] Mobile responsive
- [ ] No API keys exposed in frontend

## Cross-module contracts
- Reads Subscription (BILL-002), Usage (BILL-003)
- Calls Stripe API via service layer (BILL-001)
- Integrates with settings (FE-001)
